# 创造 — 一句话，一个世界

> 每个人心里都有一个故事，我们让它 10 秒内变成可以玩的游戏。

---

## 这是什么

用户打开网页，输入一句话：

> "一个发生在民国上海的谍战故事"

10 秒后，一个完整的文字冒险游戏出现在眼前——
有角色、有场景、有属性系统、有目标追踪、有 AI 驱动的实时对话。
不需要下载，不需要注册，不需要懂技术。打开即玩。

---

## 为什么要做这个

### 现状的痛

**创作者侧**：想做一个互动故事游戏？你需要——会写代码，会画 UI，会部署服务器。
门槛太高，99% 的好故事死在了"我不会写代码"这句话上。

**玩家侧**：想玩 AI 对话游戏？市面上的要么是千篇一律的恋爱模拟，要么是没有 UI 的纯文本聊天。
没有一个产品能让你说"我想玩一个太空歌剧"然后真的给你一个。

### 我们的机会

AI 已经能写出精彩的互动故事了。缺的不是 AI 能力，是**把故事变成游戏的那一步**。
我们有现成的 5 个游戏项目验证过的 UI 骨架。只需要在中间加一层——让 AI 把"一句话"翻译成"游戏配置"。

**一句话总结：我们要做"故事界的 Canva"——人人都能在 10 秒内创造自己的游戏世界。**

## 游戏界面 — 三栏全景

我们的核心优势是**有视觉的世界**——场景背景图、角色立绘/视频、高光时刻 AI 生图。
在此基础上，加入完整的游戏状态面板，让玩家既沉浸又有"在玩游戏"的掌控感。

│                    │

│   角色名 · 身份     │

│  👥 角色            │ ← 2x2 网格，可交互
│  ┌──┐ ┌──┐        │    灰色=未解锁（剧情推进后解锁）
│  │苏│ │陆│        │
│  └──┘ └──┘        │
│  ┌──┐ ┌──┐        │

## AI 生成什么

用户输入一句话后，AI 一次性生成以下内容：

### 1. 游戏设定
- 游戏名称（如"暗夜谍影"）
- 类型标签（如"谍战悬疑"）
- 背景描述（一段沉浸式的开场白）
- 游戏天数和每日行动力

### 2. 目标（3-4 个）
- 目标名称和描述
- 初始进度 0%
- AI 在对话过程中自动推进进度

### 3. 角色（3-5 个）
- 名字、身份、性格描写
- 主题色（自动配色，保证视觉区分度）
- 初始关系属性值
- 一个代表性的 emoji 或汉字头像
- 是否初始可见（部分角色随剧情解锁）

### 4. 场景（3-5 个）
- 场景名称、描述
- 关联哪些角色会出现
- 场景 emoji 图标

### 5. 玩家属性（3-6 个）
- 根据游戏类型自动选择属性维度
- 谍战 → 忠诚/怀疑/情报
- 恋爱 → 魅力/好感/心动
- 冒险 → 生命值/法力值/力量/智慧
- 武侠 → 内力/轻功/剑法/名望
- 每个属性的初始值、显示颜色

### 6. 物品/资产种子（2-3 个）
- 初始物品（可选）
- 定义哪些物品可以在剧情中获得
- 物品效果和属性影响

### 7. AI 叙事风格
- 根据类型生成对应的 AI 说话方式
- 谍战 → 紧张悬疑、暗流涌动
- 恋爱 → 细腻温情、氛围感强
- 恐怖 → 压迫感、留白、暗示

### 8. 主题配色
- 根据游戏氛围自动选色
- 谍战 → 深金色 + 暗红
- 恋爱 → 粉紫色
- 赛博朋克 → 霓虹青
- 武侠 → 水墨灰 + 朱红

**不生成的（现阶段）**：角色立绘图片、场景背景图片、配乐。先用 emoji/文字头像 + 纯色/渐变背景，保证核心体验流畅。后续再加 AI 生图。

## AI 对话中自动做的事

玩家每次发送消息后，AI 不只是回复故事文本，还会在回复中夹带"结构化标记"，被前端解析为游戏状态变化：

| 标记 | 作用 | 示例 |
|------|------|------|
| `【属性变化】` | 玩家数值变化 | `【忠诚 -5，情报 +10】` |
| `【目标更新】` | 目标进度推进 | `【目标更新：获取密电码 +20%】` |
| `【关键事件】` | 记录剧情里程碑 | `【关键事件：苏醒与身份确认】描述...` |
| `【解锁角色】` | 新角色出现 | `【解锁角色：藤原】` |
| `【获得物品】` | 获取新资产 | `【获得物品：如意金箍棒 · 攻击力+50】` |
| `【行动选项】` | 建议下一步操作 | `【行动选项】1. 搜查环境 2. 大声呼唤 3. 静坐冥想` |

这些标记会被前端实时解析，对应更新左栏属性、右栏目标/事件/资产、行动选项按钮。
玩家看到的是流畅的故事推进 + 随之变化的游戏面板——"一边读故事，一边玩游戏"。

## 体验打磨点

### 创造阶段

- **灵感提示**：输入框下方滚动展示示例 prompt，降低空白恐惧
  > "试试：一个发生在太空站的密室逃脱"
  > "试试：四个大学室友的毕业前最后一周"
  > "试试：古代武林，五大门派的恩怨情仇"

- **生成动画**：10 秒等待不能干等。依次展示"正在构思世界观..."、"正在创造角色..."、"正在布置场景..."、"正在规划目标..."、"正在调配色彩..."

- **一句话也能很短**：哪怕用户只输入"恐怖游戏"两个字，也能生成一个完整的世界。AI 负责补全想象力。

### 预览阶段

- 角色卡片带主题色边框，一眼看到视觉风格
- 目标列表让用户知道"这个游戏要做什么"
- 属性维度直观展示，让用户理解"这个游戏的玩法核心是什么"
- "重新生成"保留用户的原始输入，但产出不同的世界

### 游玩阶段

- 行动选项降低"不知道该干嘛"的迷茫感，但不限制自由度
- 目标进度条给玩家明确的方向感和成就感
- 关键事件弹窗制造"故事高潮"的仪式感
- 角色解锁带来"发现新世界"的惊喜
- 数值变化实时可见，每次对话都有"游戏反馈"

如果我们做对了，用户会说：

> "卧槽，我随便打了一句话，它真的给我生成了一个能玩的游戏？还有目标、有事件、有角色解锁？"

这个"卧槽时刻"就是产品的全部。

---

## 技术可行性（一段话版本）

已有 5 个游戏项目的完整 UI 骨架，正在提取通用模板。
在模板基础上，加一个"AI 世界生成器"模块——调用大模型，输入用户的一句话，输出结构化的游戏配置（JSON），注入到现有游戏框架。
AI 对话中的结构化标记（属性变化、目标更新、事件触发）复用现有的文本解析器，扩展即可。
前端纯静态，不需要后端服务器（除了 AI API 代理）。
核心技术风险为零，都是成熟方案的组合。
